## Directory Purposes

**docs/:**
- Purpose: All documentation content organized by subject area
- Contains: Markdown (.md) files with YAML frontmatter
- Key files: `intro.md` (entry point), `quick-start.md`, `reading-list.md`
- Subdirectories: `ai-engineering-cookbook/`, `engineering-habits/`, `engineering-process/`, `product/`, `people/`, `team/`, `culture/`, `tech/`, etc.

**blog/:**
- Purpose: Blog posts and time-indexed content
- Contains: Markdown files with blog post format, images, author references
- Key files: `authors.yml` (author metadata), `2021-08-26-welcome/` (folder-based post)
- Structure: Date-prefixed filenames for chronological ordering

**src/components/:**
- Purpose: Reusable React components for interactive features
- Contains: Functional components with local styling
- Key components:
  - `HomepageFeatures/` - Feature cards displayed on homepage
  - `StartupSanityForm/` - Interactive startup evaluation form with scoring
  - Each component has index file (`.jsx` or `.js`) and scoped styles (`.module.css`)

**src/pages/:**
- Purpose: Page-level components that map to URLs
- Contains: React components automatically routed by Docusaurus
- Key files: `index.js` (homepage)
- Pattern: File names map to routes (index.js → /)

**src/theme/:**
- Purpose: Custom theme component overrides
- Contains: Wrappers around Docusaurus theme components
- Key files: `DocItem/index.js` (overrides doc page rendering to inject custom meta tags)
- Pattern: Mirror Docusaurus theme structure

**src/css/:**
- Purpose: Global site styles
- Contains: CSS that applies across all pages
- Key files: `custom.css` (Infima variable overrides for branding colors)
- Pattern: Theming variables at root level

**static/:**
- Purpose: Static assets served as-is without processing
- Contains: Images (SVG, PNG), icons, favicon
- Key subdirectories: `img/` (images and logos)
- Pattern: Referenced in markdown and components using `/img/*` paths

**build/:**
- Purpose: Generated static website output
- Contains: Compiled HTML files, CSS bundles, JavaScript, optimized images
- Structure: Mirrors docs/ and pages/ structure
- Generated by: `npm run build`
- Deployment: Contents of build/ deployed to web server/CDN

**.docusaurus/:**
- Purpose: Docusaurus cache and generated metadata
- Generated by: Docusaurus during build
- Committed: No (in .gitignore)

## Naming Conventions

**Files:**
- Page components: `index.js` or `index.jsx` in route directories
- Component modules: `index.js` or `index.jsx` with accompanying styles
- CSS Modules: `*.module.css` suffix
- Documentation: lowercase with hyphens: `quick-start.md`, `intro.md`
- Blog posts: date-prefixed: `2021-08-26-welcome.md`

**Directories:**
- Components: PascalCase: `HomepageFeatures/`, `StartupSanityForm/`
- Pages: lowercase: `pages/`
- Features: lowercase with hyphens: `engineering-habits/`, `ai-engineering-cookbook/`

**Components:**
- PascalCase for component names: `HomepageFeatures`, `StartupEvaluationForm`, `Feature`
- Props: camelCase
- CSS classes: camelCase via module exports

**Variables:**
- State variables: camelCase: `responses`, `redFlags`, `result`
- Constants: camelCase (lowercase) or ALL_CAPS for configuration arrays

## Where to Add New Code

**New Documentation Page:**
- Primary code: Create `.md` file in appropriate subdirectory under `docs/`
- Example: `docs/product/new-feature.md`
- Sidebar: Auto-generated from directory structure (no manual config needed)
- Path alias: Use `docs/` path without `src/` prefix in frontmatter

**New Blog Post:**
- Primary code: Create new markdown file in `blog/` directory
- Naming: Date-prefix the filename: `blog/YYYY-MM-DD-post-title.md`
- Author metadata: Reference author from `blog/authors.yml`

**New Interactive Component:**
- Implementation: Create component directory under `src/components/`
- Structure:
  - `src/components/ComponentName/index.jsx` (component logic)
  - `src/components/ComponentName/styles.module.css` (scoped styles)
- Import: Use `@site/src/components/ComponentName` path alias in markdown or pages
- Export: Default export the component

**New Page:**
- Implementation: Create `.js` or `.jsx` file in `src/pages/`
- Naming: Lowercase name maps to route
- Pattern: Follow pattern in `src/pages/index.js` using Docusaurus hooks and Layout component

**New Theme Override:**
- Implementation: Create directory structure under `src/theme/` mirroring Docusaurus theme
- Example: `src/theme/ComponentName/index.js`
- Pattern: Import original from `@theme-original/ComponentName`, wrap with custom logic

**Utilities and Helpers:**
- Shared component logic: Extract to component-level utilities or hooks
- Currently: No shared utility folder; all logic is co-located with components
- If creating shared utilities: Create `src/utils/` directory with named exports

## Special Directories

**build/:**
- Purpose: Generated static site output
- Generated: Yes (via `npm run build`)
- Committed: No (in .gitignore)
- Deployment target: Contents deployed to Netlify or web server

**.docusaurus/:**
- Purpose: Docusaurus cache, metadata, and internal build artifacts
- Generated: Yes (via build process)
- Committed: No (in .gitignore)
- Use: Do not edit manually

**node_modules/:**
- Purpose: Installed npm dependencies
- Generated: Yes (via `npm install`)
- Committed: No (in .gitignore)
